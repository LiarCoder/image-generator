# 问题：`v2.0.0` tag 指向错误提交的修正

## 背景

- 初次补打 `v2.0.0` 后，用户确认应对齐到：
  - `1db6bf566eb542713ace1e7099a876827535bded`

## 修正动作

1. 删除本地 `v2.0.0`
2. 在指定提交重建附注标签 `v2.0.0`
3. 删除远程旧标签
4. 推送新标签到远程

## 风险提醒

- 重写已存在的远程 tag 属于“历史更正”，会影响依赖该 tag 的自动化。
- 应在团队内同步：`v2.0.0` 语义对齐规则与最终落点。

## 建议

- 后续发版前，先在本地确认 `tag -> commit`，再推送：
  - `git show <tag> --no-patch`
